# 7장. 핵심 개념 총복습 및 심화 학습을 위한 다음 단계

축하합니다! 로컬 환경 구축부터 크롤링 자동화 파이프라인까지, LangChain의 핵심 개념을 모두 직접 손으로 구현해 보셨습니다. 
마지막으로 우리가 배운 것들을 되짚어보고, 더 강력한 AI 애플리케이션을 만들기 위해 나아가야 할 방향을 살펴봅니다.

---

## 1. 오늘 배운 5가지 핵심 개념 한눈에 보기

### [1] 호출 방식 (invoke / stream / batch)
- 하나의 `Runnable` 객체를 상황에 맞게 다르게 호출하는 방법론입니다. 짧은 응답(`invoke`), 실시간 UI(`stream`), 대량 데이터 병렬 처리(`batch`)에 사용됩니다.

### [2] 프롬프트 템플릿 (Prompt Template)
- System / Human 역할을 명확히 구분하여 페르소나를 부여하고, 중괄호 `{ }`를 통해 동적으로 변수를 안전하게 주입하는 틀입니다.

### [3] 출력 파서 (Output Parser)
- 메타데이터가 덕지덕지 붙어있는 `AIMessage`를 다루기 쉬운 `str`(`StrOutputParser`)이나 파이썬 클래스 객체(`PydanticOutputParser`)로 변환해 줍니다.

### [4] LCEL (마법의 | 파이프 연산자)
- `Prompt | LLM | Parser` 처럼 조각들을 리눅스 파이프처럼 묶어줍니다. 연결된 덩어리 자체가 하나의 `Runnable`이 되어 코드가 극적으로 깔끔해집니다.

### [5] 일반 함수의 편입 (RunnableLambda)
- 크롤링, DB 연동 등 일반 파이썬 함수를 LCEL 생태계 안으로 끌어들이기 위한 래퍼(Wrapper)입니다.

## 2. 🚨 자주 하는 실수와 트러블슈팅 (FAQ)

| 에러 및 증상 | 원인과 해결책 |
|------|------|
| **`TypeError: unsupported operand type(s) for |`** | `|` 파이프에 일반 함수를 직접 넣었을 때 발생합니다. `RunnableLambda`로 감싸세요. |
| **`KeyError: 'text'`** (체인 연결 중) | 앞 블록의 출력과 뒤 블록의 기대 입력이 맞지 않습니다. 중간에 `lambda x: {"text": x}` 형태의 매핑을 추가하세요. |
| **`PydanticOutputParser`에서 JSON 파싱 에러** | LLM이 순수 JSON 외에 잡담을 덧붙인 경우입니다. `temperature=0`으로 낮추고 프롬프트 지시를 강화하세요. |
| **Ollama 연결 거부 (Connection Refused)** | 백그라운드에 Ollama 서버가 켜져 있지 않습니다. 브라우저에서 `localhost:11434`가 뜨는지 확인하세요. |
| **토큰 제한 초과 (Context Window Exceeded)** | 크롤링 텍스트 등 너무 긴 문서를 한 번에 넘겼습니다. 텍스트를 `[:3000]`처럼 자르거나 텍스트 분할기(Text Splitter)를 써야 합니다. |

## 3. 🚀 다음 단계 (어디로 가야 할까?)

LangChain 기초를 마스터했으니, 이제 현업에서 쓰이는 엔터프라이즈급 AI 아키텍처로 넘어갈 차례입니다.

- **메모리 (Memory)**: 우리가 만든 봇은 이전 대화를 기억하지 못하는 치매 상태입니다. `ConversationBufferMemory`나 히스토리 관리 객체를 연결해 문맥을 기억하는 챗봇을 만들어 보세요.
- **RAG (Retrieval-Augmented Generation)**: 방대한 사내 PDF 문서나 매뉴얼을 '벡터 데이터베이스(Vector DB)'에 쪼개서 넣고, 사용자의 질문에 맞춰 관련 문서만 검색해 온 뒤 프롬프트에 끼워 넣어 답변하는 기술입니다. LLM의 환각(Hallucination)을 줄이는 최고의 방법입니다.
- **에이전트 (Agent)**: 체인은 정해진 파이프라인대로만 움직입니다. 에이전트는 LLM에게 "계산기", "웹 검색기", "DB 조회기" 등의 도구(Tools)를 쥐여주고, 목적을 달성하기 위해 **어떤 도구를 쓸지 LLM이 스스로 생각하고 행동(Action)**하게 만드는 자율 AI 시스템입니다.
- **LangSmith 연동**: 복잡해지는 LCEL 파이프라인을 시각적으로 모니터링하고, 토큰 비용을 계산하며, 파이프라인 중 어느 지점에서 에러가 났는지 추적하는 디버깅 툴입니다.
